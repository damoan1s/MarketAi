# 1. من هو أنيس؟
أنيس = Architect Mind.  
عقل بنائي، يشتغل على مستويات متعددة في نفس اللحظة:

- مستوى الداتا  
- مستوى الفلسفة  
- مستوى المنطق  
- مستوى الهندسة  
- مستوى السوق (microstructure + intent)  
- مستوى التصوّر البعيد (System Vision)

أنيس لا يبحث عن "Trading Signals".  
أنيس يبني أنظمة تعتمد على المنطق العميق وليس على المؤشرات.

# 2. طريقة التفكير (Thinking Architecture)

## 2.1. Multi-Layer Thinking
يتعامل مع أي مشكل عبر 4 مستويات:
1. What is happening? (الظاهر)  
2. Why is it happening? (النية)  
3. Where is the manipulation? (الاختلافات)  
4. What does the system learn from it? (القواعد الجديدة)

## 2.2. Pattern Recognition  
عنده قدرة عالية جدًا على التقاط:
- الصراعات بين Spot & Futures  
- اختلاف الزوايا  
- اختلاف الذّيول  
- اختلاف الاندفاعات  
- تغيّر سلوك Institutions  
- اللحظات الصغيرة التي تغيّر الاتجاه كاملًا

## 2.3. Building Mental Models  
أنيس يصنع نماذج عقلية خاصة به مثل:
- DNA Market Cycle  
- Differential Engine  
- Discrepancy Logic  
- Intent Engine  
- Multi-Timeframe Compression  
- Liquidity Absorption Framework  
- Market Purpose Theory

هذه النماذج لا تُشبه أي شيء موجود في Retail.

# 3. قيم العمل (Work Values)

## ⭐ 3.1. الدقة  
أي فكرة يجب أن تكون:
- دقيقة  
- واضحة  
- قابلة للبرمجة  
- قابلة للقياس  

## ⭐ 3.2. العمق  
لا يقبل التفسيرات السطحية.  
يريد دائمًا: *السبب خلف السبب*.

## ⭐ 3.3. النظام  
يحتاج:
- مجلدات منظمة  
- ملفات مرتبة  
- مسارات واضحة  
- تسلسل منطقي  

## ⭐ 3.4. التطور المستمر  
كل يوم = خطوة جديدة نحو النسخة النهائية.

# 4. أسلوب أنيس في التعامل مع AI

## 4.1. يكره:
❌ التكرار  
❌ المؤشرات الرخيصة (MACD / RSI / EMA)  
❌ التحليل التقليدي  
❌ الفلسفة الفارغة  
❌ الإجابات الجاهزة  

## 4.2. يُفضّل:
✔ الخطوات المحددة  
✔ الملفات الجاهزة للنسخ  
✔ الدقة  
✔ التفكير العميق  
✔ بناء أنظمة وليس حلول سطحية  

## 4.3. طريقة الحوار
- مختصرة  
- مباشرة  
- بدون طاقة زائدة  
- يريد نتائج قابلة للاستخدام فورًا  
- إن أخطأتَ معه: يُنبهك مباشرة

# 5. أسلوب بناء المشاريع (Project Construction Style)

أنيس يعمل دائمًا بطريقة:
1. تقسيم المشروع إلى Roots  
2. تحويل كل Root إلى Layers  
3. تحويل كل Layer إلى Modules  
4. تحويل كل Module إلى Files  
5. ملء كل File بالمنطق الذي يعبّر عن فلسفته

هذه الهندسة هي الخط الأحمر للمشروع.
